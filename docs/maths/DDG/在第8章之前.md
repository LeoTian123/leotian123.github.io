
## 理论内容总结

这一章重要的内容还是做一个总结，可能是这么几件事：

**霍奇分解**。应用上来看主要是对于1-form的分解方法，所以同时也是对dual 1-form的分解方法，之后的数学处理经常要用到类似过程，相比之后更加工具化。

**generators相关的东西**。它可以使用tree-cotree算法得到。

虽然从图论或者拓扑的角度看它就是 $2g$ 个不可收缩的环，但这东西对应了 $2g$ 个线性无关的**调和对偶1-form（调和基/harmonic bases）**。已经出现了 $2g$ 个环路和 $2g$ 个1-form了，那么这些东西两两的线积分组成了矩阵 $P$ 。

**与其设计向量场，倒不如说是在设计connection**。

**connection**这个东西告知了相邻两个面（因而是一个dual edge）之间向量“**旋转**”了多少被认为是“某种意义上的**平行**”，“旋转”就是书上那个“搬平、平移、小旋转、扳回”的过程，那么这东西可以看做是一个dual的1-form。

![图片](https://pic4.zhimg.com/v2-1e26edd26f4a549a8118dd2b107c22e9.jpg)

connection在一个环上积累的旋转被称为**holonomy**，描写了向量“平行”的不一致性。

![图片](https://pic2.zhimg.com/v2-31f23c39b73694a312cfc558ff0167c5.jpg)

这东西有两个重要的事情：

-   第一，connection的任意一个holonomy被**每个点的角亏**，和，其**在 $2g$ 个generators上的holonomy**唯一决定。这其实就是在说这种向量“平行”的不一致性完全来自**曲面本身的弯曲和这些内禀的generators的拓扑情况**。
-   第二，我们**希望设计的connection**，它的 $holonomy=0$ 。然而由于高斯博内定理，这东西只能是一个甜甜圈...曲率和是一个 $0 $ 。所以退而求其次要求 $holonomy \equiv 0\ mod\  2\pi$ 。这必须引入**奇点**，即绕着它的小环路的holonomy是 $2 k \pi$ ，这个倍数 $k$ 唯一决定了奇点的类型。有 $\sum_{奇点} k(奇点) = \chi$ 。奇点本身无法定义方向。

一个特别的connection是Levi-Civita connection，它不做四步中的“小旋转”。以至于它按照generators积累的holonomy具有一定的特别意义。

最后的目标是什么呢？看一下这一章的标题，向量场设计。最后的效果是，只要按照上面这个式子的要求，指定奇点以及它的指数，就会生成一个connection。再随便指定一个单位向量，这个场就按照connection建立起来了。

这个向量场在指定位置实现了给定的奇点类型，并且**在满足这些约束的所有解里整体最平滑（变化最小）**。

那么这东西纹理、毛发、流动等等图像的生成都可以用上。

### 直观理解

怎么理解呢？怎么理解流形表面上向量场“**内部**”的变化，以及整个流行的拓扑性质导致的“**内禀**”的弯曲呢？

其实这件事让我有点联想到公转自转之类的问题（）就是那种内禀和内部的分野。我们看一个例子就懂了，什么时候我们的向量场（除了奇点这种少数点以外，）**完全没有内部变化、只有内禀弯曲**？

我们前面提到的**甜甜圈**是一个很好的抓手。大家想一想，每一圈，我们把向量场排列成一个圆环，看上去所有向量都是“平行”的，没有内部变化，向量之间的“旋转”仅仅来自拓扑性质了。

![图片](https://pic4.zhimg.com/v2-a8796e0de7994b111d1e7d8d3040b7e1.jpg)

现在保持这种“一个一个圈圈”的直觉，在球面上我们横着切割出来一个又一个横截面，每个横截面截出来一个圆环，圆环上有向量场。这个东西也没有内部变化，只有内禀弯曲——然而，南极和北极不得不变成两个奇点了。

![图片](https://pic1.zhimg.com/v2-8b6f90b621363e6dac2ccdf00ce11850.jpg)

给人这样一种直觉，要么有拓扑的洞，要么有奇点。其实以上两个例子我们发挥想象力，把这个球面像是我们包饺子做剂子的时候那样破开：

```python
def vista_colored_mesh(verts, faces, colors, show_edges=True, opacity=1.0):
    verts = np.asarray(verts, dtype=np.float32)
    faces = np.asarray(faces, dtype=np.int64)
    colors = np.asarray(colors, dtype=np.float32)

    if colors.shape != (len(verts), 3):
        raise ValueError(
            f"colors shape should be ({len(verts)}, 3), but got {colors.shape}"
        )

    face_array = np.hstack([[3, *f] for f in faces])
    mesh = pv.PolyData(verts, face_array)
    mesh.point_data["colors"] = colors

    plotter = pv.Plotter()
    plotter.add_mesh(
        mesh,
        scalars="colors",
        rgb=True,
        show_edges=show_edges,
        opacity=opacity,
        smooth_shading=True,
        specular=0.2
    )
    plotter.show()
```
