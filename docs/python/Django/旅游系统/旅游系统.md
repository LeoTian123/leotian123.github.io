# 数据结构课设设计

这里很多语句来自当时我们小组在验收表里的描述。

架构：B/S架构。前端使用Vue 3，后端使用Python 3.12下的Django 5.1.7框架。项目分为Attractions、Routing、Journals三个模块。

## 1.Attractions模块：景点推荐

基本功能：筛选、搜索、排序、个性化推荐。以下细说情况，还有实现方法。

筛选：按景点类型（景区、校园）、按照用户是否点赞。

搜索：模糊匹配关键字，给出评分。用汉字编辑距离（Levenshtein Distance）+字符编辑距离混合评分。

排序：可按照默认、热度（非动态统计）、评价（综合点赞数与日记数）进行排序，都考虑到模糊搜索的评分指标。堆选择排序算法（Heap-Based Selection Sort）。

个性化推荐：根据各个用户点赞历史推荐相似用户喜欢的景点。使用用户协同过滤（User-Based Collaborative Filtering）。


这一模块的测试数据，用requests库请求网络资源（去哪儿网），BeautifulSoup库解析，获取到大部分景点数据。共696个景点。



## 2.Routing模块：路线规划&场所查询

这一模块承担了两个主要功能。之所以这么分类也是当时课程设计的要求。

1.路线规划。完成多景点、多权重、多图论路径类型的路线规划功能。两种交通方式提供三种权重方法：最短路径、最短时间、只走路的最短时间。考虑道路拥挤度、交通工具速度等因素。

路线规划功能1：两点路线查询（路径规划算法），使用Dijkstra最短路径算法。

路线规划功能2：多景点环路路线查询（环路规划算法），使用拓扑子图数据结构+旅行商问题（TSP）的穷举算法。

2.场所查询。实现了可筛选类型的多权重场所查询功能。用户可指定当前位置、搜索个数、筛选类型、排序权重，系统返回符合条件的场所列表，按所选的权重排序。每个场所包含名称、类型、距离、预计到达时间等信息。使用Dijkstra最短路径算法+堆选择排序。


这一模块的测试数据，大部分地图使用自行设计的随机平面图生成算法，采用Delaunay三角剖分算法+生成树算法。真实地图采用自行编写的坐标采集辅助程序得到。每个地图约150节点，约700连接，建筑物、服务设施均符合要求。（这里的要求说的是当时课程设计的要求）


## 3.Journals模块：旅游日记

现了用户旅游日记的创建、查看、点赞功能，其中查看实现了筛选、搜索、排序。用户间可互相查看公开日记，进行点赞，形成旅游社区。

创建：用户可记录旅游体验，包括标题、日期、描述、图片、内容等。

筛选：查看指定景点下的日记、按照用户是否点赞、按照发布者是否为某用户。

搜索：支持对于多个关键字的搜索。使用Aho_Corasick多模式匹配算法。

排序：可按照默认、喜欢数/点赞数、时间进行排序。使用堆选择排序算法。

压缩：对于所有文本信息进行压缩存储、解压缩读取。赫夫曼编码（Huffman Coding）。这一个功能其实很鸡肋，也是当时的课程要求。


这一模块的测试数据，主要使用AI生成数据，小部分来自测试时的手动创建。
